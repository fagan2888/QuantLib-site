About SourceForge, Open Source, CVS, etc.

A C++ open source library for quantitative finance

The QuantLib project is aimed to provide a comprehensive software framework for quantitative finance. The goal is to provide a standard free/open source library to quantitative analysts and developers for modeling, trading, and risk management in real-life.

The core library is written in C++. It is also exported as a Python module. More extension modules are planned for other scripting languages, Excel/Gnumeric, MatLab/Octave, etc.

QuantLib plans to offer tools that are useful for both practical implementation, with features such as market conventions, solvers, PDEs, etc., and advanced modeling, e.g., exotic options and interest rate models.

QuantLib is meant to be used by academics and practitioners alike, eventually promoting a stronger interaction between the two.

Finance is one area where well-written open-source projects could make a tremendous difference. Almost every financial institution needs a solid, time-effective, operative implementation of leading-edge pricing models and hedging tools. However, to get there, currently one is forced to re-invent the wheel every time. Even decade-old models with no market value, such as Black-Scholes formula (1973), still lack a standard implementation. As a consequences many good quants are wasting their time writing C++ classes which have been already written thousands of times.

What would be the impact of an open-source standard financial library?

  • The library could be exploited across different research institutions, banks, software companies, and so on. Being an open-source project, quants contributing to the library would not need to start from the beginning every time he/she moves on to another institution.
  • Students could master a library that is actually used in the real world and contribute to it in a meaningful way. This would potentially place them in a privileged position on the job market.
  • Researchers would have available at hand a framework which vastly reduces the amount of low-level work necessary to build models, so that they could focus on more complex and interesting problems.
  • Small financial firms would not be forced to buy expensive software that can sometimes be more burdensome than liberating, but rather be able to engage in creating more innovative solutions that would make them more competitive on the market.

The library would be appealing to quants in any organization: academia, consulting firms, software companies, and corporate institutions. The QuantLib license grants the maximum degree of freedom in the use of the library.

RiskMap, a leading risk-management consulting firm in Italy, has agreed to commit a significant part of its resources to the development of this open-source library.

As the project is in its very early stage, the time is right to help deciding the target goals and to start contributing to the library. The project-status page gives a quick overview of the work in progress, while a complete documentation is also available.

  • Anyone interested is invited to join one of the mailing lists.
  • Check the latest news for the most recent updates on the project.
  • The released files (source code, binaries, documentation, etc.) are available at the download page.
  • The QuantLib CVS, providing all the snapshots of the development, can be accessed directly, or through the CVS web interface.

  • If you have patches you'd like to submit, the best way is to use the Patch Manager.
