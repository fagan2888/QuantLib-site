Project Status

The QuantLib project is at this time in alpha status (release 0.1.9). The next release (0.2) will mark the upgrade to beta status.

A number of C++ classes and functions are currently available. They form a library which can be linked to any external project. Also, such classes are made available as a Python module by means of SWIG. We welcome any contribution aimed at exporting the library towards other programming languages; however, the use of SWIG is to be preferred to hand-made approaches since it leads to more maintainable code. SWIG supports Java, Tcl, Perl5, Guile, MZScheme and Ruby besides Python. SWIG can be downloaded from http://swig.sourceforge.net/download.html.

Goals for release 1.0

The following targets are open for suggestions, corrections, and additions. Please consider using the QuantLib user mailing list for proposal. Every proposal will be seriously considered, especially if volunteers show up to implement it.

  1. The library core

    Interfaces
    • yield term structure
    • financial instruments
    • deals
    • folder/portfolio/book levels of deal aggregation (strictly vertical inclusion)
    • trade-group deal aggregation (horizontal cross section)
    • volatility term structure for cap/floor and swaption
    Generic Tools
    • date/time module
    • one-dimensional solver
    • one-dimensional optimizer
    • multi-dimensional solver
    • multi-dimensional optimizer
    • linear algebra module
    • PDE module
    • statistical module
    • Montecarlo module
    Financial Tools
    • Black-Scholes analytic implementation
    • Black-Scholes PDE implementation (for American and exotic options)
    • Deposit, FRA, futures, swaps
    • Deposit/FRA/futures/swap yield term structure
    • Swaption
    • Cap/floor
    • Bond (IRR/duration)
    • Treasury yield term structure (Nelson-Siegel/Vasichek-Fong)
    • Short-rate models (BDT, BK, HW, etc.)

    As you may notice this list has a fixed-income bias, so please feel free to suggest additional FX and equity targets.

    The library is object-oriented, but an additional function interface will be designed.

  2. Library documentation

    Documentation of classes and interfaces should be included in the code, automatically extracted, in multiple formats (HTML, LaTeX, PDF). As of now we're using Doxygen.

  3. Executable implementations

    • COM for porting to the Microsoft world (VB and Excel)
    • Matlab/Octave extension
    • Python module

    For porting to Python we're using SWIG. This approach may easily help porting QuantLib to Perl, Tcl, etc,

    The library should be exported to the maximum possible degree, allowing for object manipulation when possible, providing the function interface as bare minimum.

  4. Open interests

    If enough work force is available, the following items could be considered for inclusion in the goals for the 1.0 release.

    • Historical quote database (libors/swaps/equities/indexes/volatilities)
    • Simple Object Access Protocol (SOAP)
    • XML financial instrument definition (FpML?)
    • High-level documentation explaining models, assumptions, etc.
