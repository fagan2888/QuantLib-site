Project Status

The QuantLib project is at this time in alpha status (release 0.1.9). The next release (0.2) will mark the upgrade to beta status.

A number of C++ classes and functions are currently available. They form a library which can be linked to any external project.

The library core

The following list is open for suggestions, corrections, and additions. Please consider using the QuantLib-users mailing list for proposal. Every proposal will be seriously considered, especially if volunteers show up to implement it.

  1. Interfaces
    • yield term structure
    • financial instruments
    • deals
    • folder/portfolio/book levels of deal aggregation (strictly vertical inclusion)
    • trade-group deal aggregation (horizontal cross section)
    • volatility term structure for cap/floor and swaption
  2. Generic Tools
    • date/time module
    • one-dimensional solver
    • one-dimensional optimizer
    • multi-dimensional solver
    • multi-dimensional optimizer
    • linear algebra module
    • PDE module
    • statistical module
    • Montecarlo module
  3. Financial Tools
    • Black-Scholes analytic implementation
    • Black-Scholes PDE implementation (for American and exotic options)
    • Deposit, FRA, futures, swaps
    • Deposit/FRA/futures/swap yield term structure
    • Swaption
    • Cap/floor
    • Bond (IRR/duration)
    • Treasury yield term structure (Nelson-Siegel/Vasichek-Fong)
    • Short-rate models (BDT, BK, HW, etc.)

As you may notice this list has a fixed-income bias, so please feel free to suggest additional FX and equity targets.

Library documentation

Documentation of classes and interfaces is mainly included in the code, automatically extracted, in multiple formats (HTML, LaTeX, PDF) by the means of Doxygen.

QuantLib-based Extension

QuantLib is available as a Python module by means of SWIG.

SWIG supports Java, Tcl, Perl5, Guile, MZScheme and Ruby besides Python, so this approach may easily help porting QuantLib to these languages. We welcome any contribution aimed at exporting the library towards other programming languages; however, the use of SWIG is to be preferred to hand-made approaches since it leads to more maintainable code.

There is a strong interest in having QuantLib available as extension for different platform. We're looking for volunteers to help with:

  • COM for Microsoft world (VB and Excel);
  • Gnumeric add-in;
  • Matlab/Octave extension.

The library should be exported to the maximum possible degree, allowing for object manipulation when possible, providing the function interface as bare minimum.

Open interests

If enough work force is available, the following items could be considered for inclusion in the goals for the 1.0 release.

  • Historical quote database (libors/swaps/equities/indexes/volatilities)
  • Simple Object Access Protocol (SOAP)
  • XML financial instrument definition (FpML?)
  • High-level documentation explaining models, assumptions, etc.
