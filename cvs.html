QuantLib's CVS Repository

If you're new to CVS please check SourceForge documentation and our FAQ. For general (and more advanced) information consult the free CVS Book.

Quantlib's CVS repository includes four modules:

  • QuantLib (the C++ library)
  • QuantLib-Python (the Python extension)
  • QuantLib-Ruby (the Ruby extension)
  • QuantLib-site (the HTML sources of these pages)

Please note that QuantLib modules have capital Q and capital L.

You can browse the CVS tree: this gives you a great view into the current status of the project's code. You may also view the complete histories of any file in the repository.

Anonymous CVS Access

The modules can be checked out through anonymous (pserver) CVS with the following instruction set. The module you wish to check out must be specified as the modulename. When prompted for a password for anonymous, simply press the Enter key.

Unix:

export CVSROOT=:pserver:anonymous@cvs.quantlib.sourceforge.net:/cvsroot/quantlib
cvs login
cvs -z3 co -P modulename

Win32:

SET CVSROOT=:pserver:anonymous@cvs.quantlib.sourceforge.net:/cvsroot/quantlib
cvs login
cvs -z3 co -P modulename

Updates from within the module's directory do not need the CVSROOT exporting/setting.


Developer CVS Access via SSH

If you are a SourceForge-recognized QuantLib developer you can access the CVS tree via SSH. SSH1 must be installed on your client machine. Substitute modulename and developername with the proper values. Enter your site password when prompted.

Unix:

export CVSROOT=developername@cvs.quantlib.sourceforge.net:/cvsroot/quantlib
export CVS_RSH=ssh
cvs -z3 co -P modulename

Win32:

SET CVSROOT=developername@cvs.quantlib.sourceforge.net:/cvsroot/quantlib
SET CVS_RSH=ssh
cvs -z3 co -P modulename

Updates from within the module's directory do not need the CVSROOT exporting/setting.

Note for Unix users: if you use CVS, you will need some GNU tools that usually only developers use, and which are not required to build QuantLib from tarballs. These are automake, autoconf, libtool, GNU m4, GNU make, and others which might escape me now. They all come with recent GNU/Linux distributions.

To begin the build process from a CVS checkout, start with:

sh ./bootstrap

which will prepare the package for compilation. You can then use ./configure and make in the usual way.
